# R&D Agent Copilot Agent Rules

This project follows the Week1 v0.1.0 Agent backend architecture.

## 1. Project Scope

R&D Agent Copilot is an AI Agent backend system for R&D troubleshooting scenarios.

The current v0.1.0 scope includes:

- FastAPI `POST /chat`
- Rule-based Router
- Deterministic Planner
- Executor
- Local Tools
- Local RAG
- LangGraph Execution Layer
- Trace System
- Optional DeepSeek Answer Synthesizer
- Fallback answer generation

The project is currently a backend MVP. It is designed to be runnable, testable, traceable, and safe to demo without real production integrations.

## 2. Core Architecture

Every chat request must pass through this chain:

```text
User Query
-> Router
-> Planner
-> Executor
-> LangGraph Tool Nodes
-> Trace
-> Answer Synthesizer
-> Response
```

The API must not bypass Router, Planner, Executor, LangGraph tool execution, Trace, or Answer Synthesizer. Every request must return a complete in-memory trace with a UUID trace ID, stage outputs, stage latency, tool calls, synthesizer metadata, and the final answer.

## 3. Component Boundaries

Router:

- Only classifies user input as `simple_qa` or `complex_troubleshooting`.
- Must not call tools.
- Must not generate final answers.
- Must not decompose tasks or create execution plans.

Planner:

- Only creates structured plans from `query` and `route_result`.
- Must not execute tools.
- Must not generate final answers.

Executor:

- Only executes the plan generated by Planner.
- Must not re-plan.
- Must not modify Planner output.
- Must not generate final answers.

LangGraph:

- Only runs inside Executor for tool execution orchestration.
- Handles tool node conditional execution, retry, fallback, and node trace metadata.
- Must not replace Router or Planner.

Tools:

- Current tools are `log_tool`, `config_tool`, and `git_tool`.
- Must only read local deterministic sample data.
- Must not call LLMs.
- Must not call external APIs.
- Must not generate final answers.

RAG:

- Only performs local knowledge base retrieval.
- Must not generate final answers.
- Retrieval results must be recorded in `tool_results` and trace.

Answer Synthesizer:

- May use DeepSeek API when explicitly enabled and configured.
- May only generate the final Chinese answer from `route`, `plan`, `tool_results`, and `trace`.
- Must not invent facts outside tool results or retrieved knowledge.
- Must fallback when DeepSeek is disabled, missing an API key, or fails.

Trace:

- Must record `router`, `planner`, `executor`, and `synthesizer` stages.
- Executor stage must record LangGraph engine metadata, node names, tool calls, skipped nodes, retry counts, errors, and fallback information.

## 4. LLM Rules

- DeepSeek is only used for final answer generation.
- DeepSeek must not control Router.
- DeepSeek must not control Planner.
- DeepSeek must not select tools.
- Default `LLM_ENABLED=false`.
- If no API key is configured, the system must fallback.
- Tests must never make real DeepSeek API calls.

## 5. Environment Security

- Real API keys may only be stored in local `.env`.
- `.env` must be ignored by `.gitignore`.
- `.env.example` must only contain empty example values.
- Never commit a real `DEEPSEEK_API_KEY`.
- Never print a real API key in logs, test output, README, or documentation.

`.env.example` should remain:

```bash
DEEPSEEK_API_KEY=
LLM_PROVIDER=deepseek
LLM_MODEL=deepseek-v4-flash
LLM_BASE_URL=https://api.deepseek.com
LLM_ENABLED=false
```

## 6. PR Discipline

- Each PR must solve one clear goal.
- Do not mix multiple business modules in one PR.
- Do not include unrelated refactors.
- Do not push directly to `main`.
- Do not force push.
- Read `AGENTS.md` before every development task.
- Each feature branch should correspond to one PR.

## 7. Current Prohibited Actions

At the current stage:

- Do not connect a real Git API.
- Do not connect a real log platform API.
- Do not connect a real config center API.
- Do not add a database.
- Do not add Redis.
- Do not add frontend UI unless the project explicitly enters the Week2 Trace Viewer stage.
- Do not let LLMs control the tool execution chain.
- Do not commit real secrets.
